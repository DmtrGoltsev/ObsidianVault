# ADR: Мягкое удаление (Soft Delete)

## Контекст

Пользователь может удалять папки, цели и задачи. Требовалось обеспечить возможность восстановления и избежать каскадного физического удаления связанных данных.

## Решение

Использовать [[Soft_Delete|мягкое удаление]] для сущностей Folder, Goal, Task.

## Обоснование

- Пользователь может случайно удалить папку с целями и задачами — восстановление без бэкапа
- Каскадное физическое удаление папки → целей → задач необратимо
- Мягкое удаление помечает запись как неактивную (`deleted_at` timestamp), но сохраняет данные
- Фильтрация на уровне JPA/SQL: `WHERE deleted_at IS NULL`
- При совместном доступе чужие задачи не удаляются — только прекращается доступ

## Последствия

- Все запросы должны фильтровать удалённые записи
- Увеличение объёма хранимых данных (неактивные записи)
- Требуется политика очистки старых удалённых записей (TBD)

## Альтернативы

- **Физическое удаление** — отвергнуто: нет возможности восстановления
- **Архивная таблица** — отвергнута: усложняет схему и join-запросы
