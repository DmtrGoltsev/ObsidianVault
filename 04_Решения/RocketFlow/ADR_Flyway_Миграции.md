# ADR: Flyway для миграций БД

## Контекст

Требовался инструмент версионирования схемы базы данных с автоматическим применением миграций при деплое.

## Решение

Использовать [[Flyway]] для управления миграциями PostgreSQL.

## Обоснование

- Декларативные SQL-миграции — полный контроль над DDL
- Автозапуск при старте Spring Boot — исключает ручные операции при деплое
- 18 миграций (V1–V18) покрывают всю схему: пользователи, папки, цели, задачи, напоминания, устройства, sharing
- Встроенная таблица `flyway_schema_history` отслеживает применённые миграции
- Поддержка в CI: миграции тестируются через Embedded PostgreSQL (zonky)

## Последствия

- Миграции необратимы (forward-only)
- Конфликты миграций в параллельных ветках требуют ручного разрешения (переименование V-файлов)
- Все изменения схемы — строго через миграции, ручные DDL запрещены

## Альтернативы

- **Liquibase** — отвергнут: более сложный синтаксис изменений (XML/YAML/JSON), Flyway проще и ближе к нативному SQL
- **Ручные DDL** — отвергнуты: нет версионирования, риск расхождения схемы
