# ADR: Регистрация устройств через installationId upsert

## Контекст

Каждое Android-устройство должно быть зарегистрировано для получения push-уведомлений. При переустановке приложения или смене FCM-токена устройство не должно дублироваться.

## Решение

Регистрировать устройства через логический `installationId` (app-specific UUID) с upsert-семантикой: устройство с тем же installationId обновляет FCM-токен, а не создаёт новую запись.

## Обоснование

- `installationId` генерируется при первом запуске приложения и сохраняется локально
- Переустановка приложения генерирует новый installationId (предположение — зависит от реализации хранения на Android)
- FCM-токен может обновляться (Google ротирует токены) — без upsert возникают дубликаты устройств
- Upsert (INSERT ... ON CONFLICT UPDATE) гарантирует, что одна пара (user, installationId) имеет ровно одну запись
- Старые FCM-токены с истёкшим сроком автоматически очищаются Firebase

## Последствия

- При переустановке приложения старое устройство остаётся в БД (не используется, требует периодической очистки)
- Необходима уникальность installationId в рамках пользователя (гарантируется constraint)

## Альтернативы

- **Регистрация по FCM-токену** — отвергнута: токены ротируются, дубликаты неизбежны
- **Регистрация по device ID (ANDROID_ID)** — отвергнута: сбрасывается при сбросе устройства
