# Источник: API Контракты

## Путь

`docs/05-api-contracts.md`

## Роль в проекте

Определяет REST API контракты между бэкендом и клиентами (web, Android). Endpoints, форматы запросов/ответов, коды ошибок.

## Краткое содержание

Спецификация API по модулям: auth (регистрация, вход), folders/goals/tasks (CRUD), sharing (приглашения, управление доступом), calendar (расписание), reminders/recurrence (правила), notifications (отправка, статус). Соглашения по именованию и обработке ошибок.

## Статус актуальности

Актуален. Клиенты (React, Android) реализуют API согласно этому документу.
