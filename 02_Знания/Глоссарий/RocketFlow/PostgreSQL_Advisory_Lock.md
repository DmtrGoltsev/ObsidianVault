# PostgreSQL Advisory Lock

## Определение

Механизм PostgreSQL для прикладных блокировок на уровне сессии или транзакции. Используется в RocketFlow для координации конкурентных операций в рамках [[Modular_Monolith|модульного монолита]].

## Контекст в проекте

- Используется для предотвращения гонок при планировании задач и отправке уведомлений (предположение)
- Позволяет избежать дублирования напоминаний при параллельных запросах (предположение)
- Работает на уровне [[PostgreSQL_Advisory_Lock|PostgreSQL]], не требует внешнего координатора

## Бизнес-правила

- Advisory lock освобождается при завершении транзакции или сессии (предположение)
- Применяется выборочно, не на все операции (предположение)
