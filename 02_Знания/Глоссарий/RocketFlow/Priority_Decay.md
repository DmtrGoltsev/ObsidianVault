# Priority Decay (Снижение приоритета)

## Определение

Автоматическое снижение приоритета [[Task|задачи]] после повторных переносов. Контролируется пользовательскими настройками и разделено по типу задачи.

## Бизнес-правила

- Активируется при [[Reschedule_Event|переносе]] задачи
- [[Green_Task|Зелёные]] и [[Red_Task|красные]] задачи имеют раздельные политики decay
- Настройки хранятся в User Settings (источник: `docs/03-domain-specification.md`)
- Приоритет не может выйти за границы 1–10
- Модуль бэкенда: `prioritypolicy` (источник: `docs/04-architecture-blueprint.md`)

## Пример использования

- Пользователь 3 раза перенёс задачу — приоритет автоматически снижен с 7 до 4
- Для красных задач decay отключён — приоритет не снижается

## Связанные термины

- [[Task]] — задача с приоритетом
- [[Reschedule_Event]] — триггер снижения
- [[Green_Task]] — отдельная политика decay
- [[Red_Task]] — отдельная политика decay
