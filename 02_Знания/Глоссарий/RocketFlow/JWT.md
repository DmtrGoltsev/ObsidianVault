# JWT (JSON Web Token)

## Определение

Механизм аутентификации, используемый в RocketFlow: клиент получает токен от бэкенда и передаёт его в заголовках последующих запросов к [[REST_API]].

## Контекст в проекте

- Spring Security настроен на JWT-аутентификацию
- Защищает endpoints от неавторизованного доступа
- Контроллер `AuthController.java` (`backend/src/main/java/com/rocketflow/auth/AuthController.java`) выдаёт токены при входе
- Токен содержит идентификатор пользователя и срок действия (предположение)

## Бизнес-правила

- Токен обязателен для всех защищённых endpoints
- При истечении срока действия требуется повторный вход (предположение)
- Токены не должны сохраняться в коде или логах (секрет)
