# Red Task (Красная задача)

## Определение

Задача, невыполнение которой ведёт к негативным последствиям. Тип `red` в доменной модели [[Task]].

## Бизнес-правила

- Если выполнена — результат в основном сохранение текущего состояния
- Если не выполнена — состояние ухудшается
- Имеет собственные настройки [[Priority_Decay|снижения приоритета]], отдельные от зелёных задач (источник: `docs/03-domain-specification.md`)

## Пример использования

- «Оплатить счёт» — красная задача: невыполнение ведёт к штрафу
- «Записаться к врачу» — красная задача: невыполнение = риск для здоровья

## Связанные термины

- [[Task]] — родительская сущность
- [[Green_Task]] — противоположный тип
- [[Priority_Decay]] — политика снижения приоритета
