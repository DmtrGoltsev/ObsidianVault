# Task Link (Связь задач)

## Определение

Логическая связь между двумя задачами. Показывает, что задачи связаны, даже если это разные рабочие единицы.

## Бизнес-правила

- Связи только ассоциативные в MVP
- Не создают блокирующего поведения выполнения
- Используются для группировки и навигации
- В MVP движок зависимостей не вводится

## Поля

- id, source_task_id, target_task_id, link_type (опционально)

## Пример использования

- Задача «Подготовить план» связана с задачей «Согласовать план с руководителем»
- Задача «Купить билеты» связана с задачей «Забронировать отель»

## Связанные термины

- [[Task]] — исходная и целевая задача связи
- [[Goal]] — контекст, в котором находятся связанные задачи
