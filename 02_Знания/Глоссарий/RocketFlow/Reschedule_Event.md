# Reschedule Event (Событие переноса)

## Определение

Сохранённая запись о том, что [[Task|задача]] была отложена или перенесена на более позднее время.

## Бизнес-правила

- Фиксирует факт переноса задачи
- Содержит старые и новые значения [[Planned_Time|планового времени]]
- Влияет на [[Priority_Decay|снижение приоритета]] при повторных переносах
- Хранится как событие аудита для аналитики поведения пользователя

## Связанные термины

- [[Task]] — задача, которая была перенесена
- [[Planned_Time]] — плановое время (изменяемое поле)
- [[Priority_Decay]] — механизм, активируемый переносами
- [[Recurrence_Rule]] — повторение (перенос может влиять на паттерн)
