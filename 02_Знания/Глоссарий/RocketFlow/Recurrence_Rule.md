# Recurrence Rule (Правило повторения)

## Определение

Правило, описывающее, как [[Task|задача]] повторяется во времени.

## Бизнес-правила

- Связано с конкретной задачей
- В MVP задача может иметь только одно правило повторения
- При срабатывании правила создаётся новый экземпляр задачи (или обновляется существующий) (предположение)
- Модуль бэкенда: `recurrence` (источник: `docs/04-architecture-blueprint.md`)

## Пример использования

- «Совещание каждый понедельник в 10:00»
- «Оплата счетов каждое 1-е число месяца»

## Связанные термины

- [[Task]] — задача с повторением
- [[Reminder_Rule]] — правило напоминания (отдельная сущность)
- [[Reschedule_Event]] — событие переноса (может влиять на повторение)
