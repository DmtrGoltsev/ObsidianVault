# Modular Monolith (Модульный монолит)

## Определение

Архитектурный стиль, выбранный для RocketFlow: одно бэкенд-приложение с чёткими границами модулей, одной реляционной БД и одним фоновым процессом планирования внутри рантайма.

## Обоснование выбора (из `docs/04-architecture-blueprint.md`)

- Домен MVP достаточно богат для сильных границ модулей
- Команда не выигрывает от сложности распределённых систем на данном этапе
- Один деплоимый бэкенд проще запускать, отлаживать, тестировать и развивать
- Систему можно разделить позже при реальном давлении масштабирования

## Модули бэкенда

`auth`, `accounts`, `settings`, `folders`, `goals`, `tasks`, `sharing`, `calendar`, `recurrence`, `reminders`, `prioritypolicy`, `notifications` (источник: `docs/04-architecture-blueprint.md`)
