# Red Task (Красная задача)

## Определение

Красная задача — это задача-индикатор прокрастинации. Назначается системой автоматически, когда задача просрочена (дедлайн прошёл, а задача не завершена) или когда пользователь длительное время не взаимодействовал с задачей. Красный цвет сигнализирует: «Это нужно сделать, иначе приоритет продолжит падать».

## Свойства

- Назначается автоматически системой (серверный планировщик)
- Отображается с красным индикатором в списке задач
- Имеет собственный коэффициент decay (см. [[Priority_Decay]])
- Снимается автоматически при завершении задачи или переносе дедлайна
- Стимулирует пользователя к действию через визуальный сигнал

## Отличие от зелёной задачи

Зелёная задача ([[Green_Task]]) — осознанный выбор пользователя отметить задачу как важную. Красная задача — автоматический индикатор, назначаемый системой при просрочке.
