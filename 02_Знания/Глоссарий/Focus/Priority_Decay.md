# Priority Decay (Автоснижение приоритета)

## Определение

Priority Decay — это механизм автоматического снижения приоритета задачи с течением времени. Реализован в модуле `prioritypolicy` бэкенда. Выполняется фоновым планировщиком по расписанию. Для зелёных и красных задач используются раздельные коэффициенты decay.

## Принцип работы

1. Планировщик периодически (например, раз в час) сканирует незавершённые задачи
2. Для каждой задачи вычисляется время с момента создания/последнего изменения
3. Приоритет снижается по формуле: `new_priority = base_priority * decay_factor ^ time_periods`
4. Зелёные задачи ([[Green_Task]]) используют коэффициент `green_decay_factor`
5. Красные задачи ([[Red_Task]]) используют коэффициент `red_decay_factor`
6. Задача не может уйти ниже минимального порога приоритета

## Коэффициенты

- `green_decay_factor` — более медленное снижение (задача важна осознанно)
- `red_decay_factor` — более быстрое снижение (стимулирует к действию)

## Обоснование

Раздельные коэффициенты для зелёных и красных задач обоснованы в [[ADR_Priority_Decay]].
