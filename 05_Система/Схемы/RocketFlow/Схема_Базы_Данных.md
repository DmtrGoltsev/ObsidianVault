# Схема Базы Данных

## Обзор

Реляционная схема RocketFlow на PostgreSQL 16. Миграции управляются [[Flyway]]. Точная структура — в `backend/src/main/resources/db/migration/`.

## Основные таблицы (на основе доменных сущностей)

| Таблица | Сущность | Ключевые поля |
|---------|----------|--------------|
| `users` | Пользователь | id, email, display_name, timezone, created_at, active |
| `user_credentials` | Учётные данные | user_id, password_hash, password_updated_at |
| `user_settings` | Настройки | user_id, language, green_decay_policy, red_decay_policy, notification_prefs |
| `folders` | [[Folder]] | id, owner_user_id, name, description, display_order, archived |
| `goals` | [[Goal]] | id, folder_id, owner_user_id, name, description, archived |
| `tasks` | [[Task]] | id, goal_id, owner_user_id, title, description, type, priority, status, planned_time, due_time, completed_at, archived, version |
| `tags` | [[Tag]] | id, owner_user_id, name, color |
| `task_tags` | Связь Task-Tag | task_id, tag_id |
| `task_links` | [[Task_Link]] | id, source_task_id, target_task_id, link_type |
| `goal_shares` | Доступ к цели | id, goal_id, owner_user_id, collaborator_user_id |
| `recurrence_rules` | [[Recurrence_Rule]] | id, task_id, rule_type, interval, days_of_week и др. (предположение) |
| `reminder_rules` | [[Reminder_Rule]] | id, task_id, offset_minutes, trigger_type (предположение) |
| `reschedule_events` | [[Reschedule_Event]] | id, task_id, old_planned_time, new_planned_time, created_at |
| `share_invitations` | [[Share_Invitation]] | id, goal_id, owner_user_id, invitee_email, status |
| `device_registrations` | [[Device_Registration]] | id, user_id, fcm_token, device_info, registered_at |
| `notification_deliveries` | [[Notification_Delivery]] | id, task_id, device_registration_id, status, sent_at |

## Паттерны

- **[[Soft_Delete]]:** флаг `archived` в folders, goals, tasks
- **[[Optimistic_Locking]]:** поле `version` в задачах и других конкурентных сущностях (предположение)
- **[[PostgreSQL_Advisory_Lock]]:** для координации планировщика (предположение)

## Индексы (предположение)

- `users(email)` — уникальный
- `tasks(goal_id, status)` — выборка задач цели по статусу
- `tasks(owner_user_id, planned_time)` — планирование по пользователю
- `device_registrations(user_id)` — поиск устройств пользователя

## Примечание

Точная схема — в Flyway миграциях. Данная заметка — реконструкция на основе доменной модели. `уверенность: средняя`.
