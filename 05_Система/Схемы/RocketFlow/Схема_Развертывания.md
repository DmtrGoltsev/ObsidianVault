# Схема Развертывания

## Обзор

Схема развёртывания RocketFlow MVP: один production-сервер [[HexCore]] с backend jar, web статикой и PostgreSQL 16.

## Компоненты

```
GitHub (репозиторий + Actions)
    │
    ├── CI: backend-verify, web-verify, android-verify
    │
    └── CD: backend-hexcore-prod-deploy
            │
            ▼
    ┌─────────────────────────────────────┐

    │                                      │
    │  ┌──────────────────────────────┐   │
    │  │  Nginx (:443)                 │   │
    │  │  └── web статика (React SPA)  │   │
    │  └──────────────────────────────┘   │
    │                                      │
    │  ┌──────────────────────────────┐   │
```

## Потоки

1. **Web-пользователь** → HTTPS → Nginx → статика (React SPA) → API-запросы к бэкенду (:8080)
2. **Android-пользователь** → API-запросы напрямую к бэкенду (:8080) через HTTPS
3. **Бэкенд → FCM** → push-уведомление → Android-устройство
4. **CI/CD → HexCore** → деплой jar и статики через GitHub Actions

## Системные зависимости

- systemd unit: `rocketflow-backend.service`
- Nginx: reverse proxy + раздача статики
- PostgreSQL 16: локально на HexCore (предположение)
- FCM: внешний сервис Firebase
