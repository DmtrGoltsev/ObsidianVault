# Регламент Git-изоляции агентов

Полная инструкция по работе с GitHub: [[Регламент_GitHub_для_Агентов]]

## Главные запреты

1. Любые изменения кода, документации, Obsidian vault или конфигурации делай только в отдельной Git-ветке.
2. Никогда не изменяй напрямую ветки `main`, `master`, `develop`, `release/*`.
3. Никогда не выполняй без явного разрешения:
   - `git merge`
   - `git rebase`
   - `git push` (в основную ветку)
   - `git push --force`
   - `git reset --hard`
4. Merge в основную ветку выполняет только оркестратор вручную после проверки.

## Preflight — перед началом работы

Выполни и проверь:

```bash
git status                        # это Git-репозиторий?
git branch --show-current         # какая ветка?
git status --short                # чужие незакоммиченные изменения?
```

**Стоп-правила:**
- Если текущая ветка `main`, `master`, `develop` или `release/*` → создай новую ветку до любых изменений
- Если есть чужие незакоммиченные изменения → не трогай их, остановись, спроси

## Именование веток

Формат: `agent/<среда>/<задача>`

| Компонент | Примеры |
|-----------|---------|
| `<среда>` | `opencode`, `deepseek`, `codex`, `claude` |
| `<задача>` | `task-tracker-plan`, `android-task-steps`, `obsidian-memory-rules` |

Примеры:
- `agent/opencode/task-tracker-plan`
- `agent/deepseek/android-task-steps`
- `agent/codex/obsidian-memory-rules`

## Параллельная работа агентов

- Каждый агент работает в своей ветке.
- При возможности используй отдельный `git worktree`:
  ```bash
  git worktree add ../worktree-agent2 agent/opencode/task-2
  ```
- Не редактируй файлы, которые уже изменяет другой агент (если задача явно не про интеграцию).

## Порядок работы

1. Создай или переключись на отдельную ветку.
2. Сделай изменения только в рамках задачи.
3. Запусти доступные проверки/тесты.
4. Не мержи изменения в основную ветку.
5. Верни отчёт:

```markdown
- **Ветка:** agent/opencode/my-task
- **Изменённые файлы:** file1.md, file2.ts
- **Запущенные проверки:** npm test, cargo check
- **Результат:** пройдено / ошибки
- **Осталось:** что ещё нужно сделать
- **Конфликты:** есть / нет
```
