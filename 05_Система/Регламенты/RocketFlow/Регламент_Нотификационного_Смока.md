# Регламент Нотификационного Смока

## Цель

Стандартизировать процедуру smoke-тестирования канала нотификаций RocketFlow: от бэкенда через [[FCM]] до Android-устройства.

## Область применения

- Локальное тестирование нотификаций
- Staging certification
- Регрессионная проверка после изменений в модулях `notifications`, `reminders`

## Процедура

1. **Подготовка окружения**
   - Запустить бэкенд (Spring Boot, dev-профиль)
   - Убедиться, что PostgreSQL доступен
   - Убедиться, что FCM credentials настроены (без хранения секретов)

2. **Создание тестовых данных**
   - Создать задачу с [[Reminder_Rule]]
   - Установить [[Planned_Time]] на ближайшее время
   - Зарегистрировать тестовое устройство через [[Device_Registration]]

3. **Ожидание триггера**
   - Дождаться срабатывания `ReminderNotificationScheduler`
   - Проверить логи бэкенда на отправку FCM

4. **Проверка push на устройстве**
   - Убедиться, что уведомление получено на Android
   - Tap по уведомлению → открытие задачи

5. **Верификация**
   - Запись о доставке в `notification_deliveries`
   - Статус задачи корректен

## Участники

- [[Агент_QA]] — исполнитель smoke-теста
- [[Агент_Бэкенд]] — подготовка бэкенда
- [[Агент_Android]] — проверка на устройстве

## Критерии успеха

- Цепочка `reminder → push → tap → task open` работает
- 3 последовательных успешных прогона без ошибок

## Эскалация

Отказ smoke → проверка логов FCM → проверка network между бэкендом и FCM → [[Агент_Бэкенд]] / [[Агент_DevOps]] → [[Оркестратор]].
