# Регламент: Добавление нового проекта в Obsidian vault

При появлении нового проекта агент обязан выполнить шаги ниже. Vault: `%USERPROFILE%\Documents\ObsidianVault`.

## Шаг 1. Создать проектные подпапки

```powershell
$v = "$env:USERPROFILE\Documents\ObsidianVault"
$p = "ИмяПроекта"
@(
    "$v\01_Работа\Проекты\$p",
    "$v\01_Работа\Задачи\$p",
    "$v\02_Знания\Глоссарий\$p",
    "$v\02_Знания\Источники\$p",
    "$v\02_Знания\Доказательства\$p",
    "$v\02_Знания\Пакеты_контекста\$p",
    "$v\03_Агенты\$p",
    "$v\04_Решения\$p",
    "$v\05_Система\Карты_MOC\$p",
    "$v\05_Система\Регламенты\$p",
    "$v\05_Система\Схемы\$p",
    "$v\05_Система\Промпты\$p"
) | ForEach-Object { New-Item -ItemType Directory -Path $_ -Force }
```

## Шаг 2. Создать главную заметку проекта

Файл: `01_Работа/Проекты/ИмяПроекта/ИмяПроекта.md`

Использовать [[Шаблон_Проект]]. Заполнить: id, цель, стек, статус, команда, окружения, риски. Добавить `[[MOC_Все_Проекты]]`.

## Шаг 3. Создать MOC-карту проекта

Файл: `05_Система/Карты_MOC/ИмяПроекта/MOC_ИмяПроекта.md`

## Шаг 4. Зарегистрировать проект в MOC_Все_Проекты

Открыть `05_Система/Карты_MOC/MOC_Все_Проекты.md`, добавить в секцию «Активные проекты».

## Шаг 5. Создать задачи (опционально)

Файлы в `01_Работа/Задачи/ИмяПроекта/`. Использовать [[Шаблон_Задача]].

## Шаг 6. Создать пакеты контекста (опционально)

Файлы в `02_Знания/Пакеты_контекста/ИмяПроекта/`. Использовать [[Шаблон_Пакет_Контекста]].

## Шаг 7. Добавить ADR (опционально)

Файлы в `04_Решения/ИмяПроекта/`. Использовать [[Шаблон_Решение]].

## Шаг 8. Пополнить глоссарий (опционально)

Файлы в `02_Знания/Глоссарий/ИмяПроекта/`. Использовать [[Шаблон_Термин]].

## Шаг 9. Создать источники (опционально)

Файлы в `02_Знания/Источники/ИмяПроекта/`. Использовать [[Шаблон_Источник]].

## Шаг 10. Создать агентов (опционально)

Файлы в `03_Агенты/ИмяПроекта/`. Использовать [[Шаблон_Агент]].

## Шаг 11. Создать доказательства (опционально)

Файлы в `02_Знания/Доказательства/ИмяПроекта/`. Использовать [[Шаблон_Доказательство]].

## Правила

- Язык: русский (тех. названия — английский)
- YAML frontmatter обязателен
- Код не копировать — ссылки на файлы репозитория
- `[[Ссылки]]` на связанные заметки
- Секреты не сохранять
- Уверенность без источников — `низкая`
