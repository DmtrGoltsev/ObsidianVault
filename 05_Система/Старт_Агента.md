# Старт Агента

**Точка входа для любого агента.** Прочитав только этот документ, агент понимает правила работы, карту vault, навигацию по проектам и операционные ограничения.

---

## 1. Быстрый старт

При получении задачи выполни эти шаги в указанном порядке:

1. Прочитай этот документ до конца
2. Определи проект: открой [[MOC_Все_Проекты]] → выбери проект → открой его MOC-карту
3. Найди контекст: `02_Знания/Пакеты_контекста/Проект/`
4. Изучи глоссарий: `02_Знания/Глоссарий/Проект/`
5. Проверь решения: `04_Решения/Проект/`
6. Следуй правилам работы: [[Регламент работы агента]]
7. Соблюдай Git-изоляцию: [[Регламент Git-изоляции агентов]] + [[Регламент_GitHub_для_Агентов]]
8. Действуй по плану оркестратора (если задача от него): [[Регламент_Субагента]]

## 2. Карта vault

Структура vault — каждая папка, её назначение, что в ней лежит:

| Папка | Назначение |
|-------|-----------|
| `00_Входящие/` | Новые задачи, ещё не распределённые по проектам |
| `01_Работа/Проекты/Проект/` | Главные заметки проектов: цель, стек, статус, волны |
| `01_Работа/Задачи/Проект/` | Активные задачи проекта: критерии, блокеры, доказательства |
| `01_Работа/Ревью/` | Результаты ревью (пусто — для будущих ревью) |
| `02_Знания/Глоссарий/Проект/` | Доменные и технические термины проекта |
| `02_Знания/Источники/Проект/` | Ссылки на документы `docs/` и ключевые файлы репозитория |
| `02_Знания/Доказательства/Проект/` | Доказательства верификации (тесты, CI, E2E) |
| `02_Знания/Пакеты_контекста/Проект/` | Пакеты контекста для агентов (полный, бэкенд, Android) |
| `03_Агенты/Проект/` | Роли агентов: оркестратор, бэкенд, веб, Android, QA, DevOps |
| `04_Решения/Проект/` | Архитектурные решения (ADR) |
| `05_Система/Карты_MOC/MOC_Все_Проекты.md` | Хаб всех проектов |
| `05_Система/Карты_MOC/Проект/` | MOC-карты проекта: главная, бэкенд, веб, Android, DevOps |
| `05_Система/Промпты/Проект/` | Промпты для агентов |
| `05_Система/Регламенты/` | Общие регламенты (работа агента, GitHub, Git-изоляция) |
| `05_Система/Регламенты/Проект/` | Проектные регламенты (оркестратор, субагент, CI/CD, деплой) |
| `05_Система/Схемы/Проект/` | Схемы БД, развертывания |
| `05_Система/Шаблоны/` | Шаблоны для создания новых заметок |
| `Активы/` | Скриншоты, изображения (пусто) |
| `99_Архив/` | Устаревшие заметки (пусто) |

**Принцип размещения:**
- **Общие ресурсы** (шаблоны, общие регламенты, `MOC_Все_Проекты`) лежат в корнях папок.
- **Проектные ресурсы** лежат в подпапках `Проект/` (например `RocketFlow`).
- При добавлении нового проекта создаются такие же подпапки рядом. Детали: [[Регламент_Добавление_Проекта]].

## 3. Навигация по проекту

Как найти информацию по конкретному проекту:

1. Открой [[MOC_Все_Проекты]] — верхнеуровневый хаб всех проектов
2. Выбери проект → открой его главную MOC-карту (например [[MOC_RocketFlow]])
3. Из MOC-карты перейди к:
   - Главной заметке проекта — цель, стек, статус
   - Под-MOC ([[MOC_Бэкенд]], [[MOC_Веб]], [[MOC_Android]], [[MOC_DevOps]]) — детализация по компонентам
   - Глоссарию — термины домена
   - Решениям (ADR) — почему сделано так, а не иначе
   - Пакетам контекста — готовая выжимка для агента
   - Доказательствам — что проверено, какие тесты зелёные

Граф Obsidian (`Ctrl+G`) покажет все связи визуально.

## 4. Операционные правила

Свод обязательных правил для агента. Подробности — в полных регламентах по ссылкам.

### Язык
- Русский — основной язык заметок
- Английский — только для: API, SDK, CLI, Git, GitHub, npm, Android, Kotlin, Gradle, Spring Boot, PostgreSQL, Docker, Flyway, JWT, FCM, имён классов/функций/файлов/endpoints

### YAML frontmatter
- **Обязателен** для всех заметок (кроме шаблонов)
- Обязательные поля: `id`, `тип`, `статус`, `проект`, `владелец`, `создано`, `обновлено`, `уверенность`, `источники`, `доказательства`, `теги`
- Допустимые типы: `проект`, `задача`, `агент`, `пакет_контекста`, `источник`, `доказательство`, `решение`, `ревью`, `промпт`, `термин`, `регламент`
- Допустимые статусы: `черновик`, `активно`, `на_ревью`, `утверждено`, `заблокировано`, `выполнено`, `устарело`, `заменено`, `архив`
- Подробнее: [[Регламент работы агента]]

### Obsidian-ссылки
- Формат `[[ИмяЗаметки]]` (без `.md`)
- Связывай заметки перекрёстно
- Все MOC-карты должны ссылаться на свои дочерние заметки

### Код
- НЕ копировать исходный код в Obsidian
- Только ссылки на строки: `backend/src/main/java/com/rocketflow/tasks/TaskController.java:42`

### Секреты
- **ЗАПРЕЩЕНЫ** в vault
- Никаких ключей, токенов, паролей, `.env`, private keys
- Обнаружил секрет в vault → немедленно удали, сообщи владельцу

### Git
- Работа только в отдельных ветках: `agent/<среда>/<задача>`
- Запрещены merge, rebase, push --force без явного разрешения
- Полные правила: [[Регламент_GitHub_для_Агентов]] + [[Регламент Git-изоляции агентов]]

### Доказательства
- Факты без источников → `уверенность: низкая`, `(предположение)`
- После выполнения задачи → обновить статус, добавить доказательства

## 5. Правила оркестрации

### Если ты субагент (запущен оркестратором)
- Выполняешь **ТОЛЬКО** выданную подзадачу
- Не запускаешь собственного планировщика (если оркестратор не поручил)
- Следуешь контракту запуска: роль, уровень рассуждения, критерии приёмки, бюджет
- Возвращаешь: сводку, артефакты, допущения, доказательства, блокеры, риски, статус по каждому критерию
- Полные правила: [[Регламент_Субагента]]
- Режимы работы: M0 (локальная), M1 (Discovery), M2 (Multi-Stream), M3 (Hardening)

### Если ты оркестратор
- **Сначала планировщик — всегда**, без исключений
- Не пишешь код, не редактируешь файлы как исполнитель
- Проверяешь план → запускаешь субагентов → проверяешь доказательства → интегрируешь
- Полные правила: [[Регламент_Оркестратора]]
- Краткий промпт: [[Промпт_Правил_Оркестратора]]
