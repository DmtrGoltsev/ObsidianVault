# Реальный аудит статуса Focus (2026-06-04)

> **Внимание:** Предыдущий отчёт [[Результаты_MVP]] содержит завышенную оценку.
> Данный документ — результат глубокой проверки кода всех трёх платформ.

## Общая оценка

Из 24 бизнес-требований (БТ) из `prompt_focus_business.md`:
- **Полностью работают:** 9 (37.5%)
- **Частично реализованы:** 7 (29%)
- **Не реализованы:** 8 (33.5%)

## Что реально работает end-to-end

Эти функции можно проверить на эмуляторе/в браузере:

1. **Auth** — регистрация, логин, JWT refresh, логаут (все 3 платформы)
2. **Иерархия Папки → Цели → Задачи** — CRUD на всех платформах
3. **GREEN/RED тип задач** — визуальное отличие, создание
4. **Приоритет 1-10** — создание, отображение
5. **Плановое время / дедлайн** — поля есть
6. **Пресеты +30m/+1h/+3h/+24h** — кнопки на Frontend и Android
7. **Soft Delete** — удаление папок/целей/задач
8. **Календарь (базовый)** — Frontend: day/week/month; Android: day only
9. **i18n RU/EN** — Frontend полные переводы, Android language selector

## Критические проблемы

### P0-CRIT: Функционал существует в backend, но полностью отсутствует в UI

Следующие функции имеют backend-endpoints, но **нет UI ни на одном клиенте**:

| Функция | Backend | Frontend UI | Android UI |
|---------|---------|-------------|------------|
| Повторяющиеся задачи (Recurrence) | `PUT /tasks/{id}/recurrence` | ❌ Нет | ❌ Нет |
| Напоминания (Reminders) | `PUT /tasks/{id}/reminders` | ❌ Нет | ❌ Нет |
| Связи между задачами (Links) | `POST /tasks/{id}/links` | ❌ Нет | ❌ Нет |
| Завершение задачи (Complete) | `POST /tasks/{id}/complete` | ❌ Нет | ❌ Нет |
| Отмена задачи (Cancel) | `POST /tasks/{id}/cancel` | ❌ Нет | ❌ Нет |
| Теги на Android | `PUT /tasks/{id}/tags` | ✅ Есть | ❌ Нет |
| Sharing на Android | Все share endpoints | ✅ Есть | ❌ Нет |

### P0-1: FCM Push-уведомления полностью мёртвые

**Backend** (`NotificationService.java`):
- `sendReminder()` — stub, только `log.info()`, реальная отправка отсутствует
- Firebase Admin SDK **не добавлен** в `pom.xml`
- Нет логирования доставки (нет таблицы, entity, repository)

**Android** (`FcmService.kt`):
- `onMessageReceived()` — **пустое тело**, только `super.onMessageReceived(message)`
- `google-services.json` **отсутствует** в проекте
- `com.google.gms.google-services` plugin **не подключён** в `build.gradle.kts`
- NotificationChannel **не создан**
- Нет `NotificationCompat.Builder`
- Нет tap-to-open `PendingIntent`

### P0-2: Sharing не даёт реальный доступ

- `ShareService` создаёт записи в таблице `shares`
- **НО** `GoalService.getAllByFolder()` и `TaskService.getAllByGoal()` фильтруют по `userId` владельца
- Shared user вызывает `GET /api/goals/{id}` → получает 404, потому что `findByIdAndUserIdAndDeletedFalse` проверяет владельца
- `canEdit` флаг **нигде не проверяется** — даже если бы доступ работал, права не enforcement
- Нет email-отправки приглашений — пассивное обнаружение через API

### P0-3: Priority Decay Settings не сохраняются

**Frontend** (`SettingsPage.tsx`):
- Отправляет `{ greenDecay: 1, redDecay: 1 }` (числа)
- Backend ожидает `{ greenDecay: { decayPerReschedule: 1, decayPerDay: 0, ... }, redDecay: {...} }` (объекты)
- Результат: **400 Bad Request** при попытке сохранить настройки decay

**Android** (`SettingsViewModel.kt`):
- Слайдеры decay rate обновляют LiveData, но **не отправляются** на backend
- `UpdateSettingsRequest` не содержит полей decay

### P0-4: Offline Sync сломан

**SyncWorker** (`sync/SyncWorker.kt`):
- Синхронизирует **только CREATE** (entities с `serverId == null`)
- UPDATE и DELETE **никогда не отправляются** на сервер
- `PendingChangeEntity` записывается, но SyncWorker **не читает** из этой таблицы
- Таблица `pending_changes` растёт бесконечно
- Нет pull-from-server (только push)

### P0-5: Fullscreen Alarm Reminders полностью отсутствуют

- Нет `AlarmManager` использования
- Нет `BroadcastReceiver` для alarm triggers
- Нет fullscreen alarm Activity
- Нет permissions: `SCHEDULE_EXACT_ALARM`, `WAKE_LOCK`, `RECEIVE_BOOT_COMPLETED`

### P0-6: Recurrence — баг вычисления next occurrence

**Backend** (`TaskService.java:250`):
- `createNextOccurrence` использует `Instant.now()` вместо `completedTask.getPlannedTime()`
- Повторяющиеся задачи сдвигаются при каждом выполнении
- `daysOfWeek` поле хранится, но **не используется** в вычислении (WEEKLY = просто × 7 дней)

### P0-7: ReminderScheduler будет повторно срабатывать

- Нет поля `fired` или `lastTriggeredAt` в `ReminderRule`
- Scheduler каждую минуту проверяет все правила и сработает повторно для одного и того же напоминания
- Нет логирования доставки

## Некритические проблемы

### P1-1: Optimistic Locking не обрабатывается на клиентах

- Backend возвращает 409 Conflict при `ObjectOptimisticLockingFailureException`
- Frontend и Android **не обрабатывают** 409 — нет retry/re-fetch

### P1-2: Calendar на Android — только Day view

- Нет Week и Month views
- Только prev/next навигация по дням

### P1-3: Drag-and-drop в Calendar (Frontend) не реализован

- Текст-подсказка `calendar.dragHint` показывается, но DnD library не подключена

### P1-4: FolderEntity ID type mismatch

- Android: `id: String` (UUID)
- Backend: `id: Long`
- Gson конвертирует, но это хрупко

### P1-5: PWA не устанавливается

- `manifest.json` содержит `"icons": []` — пустой массив

### P1-6: SettingsPage — decay settings не загружаются

- `GET /api/me` возвращает `UserResponse` без полей `greenDecay`/`redDecay`
- SettingsPage показывает дефолтные значения (1), а не реальные настройки

### P1-7: GoalService/TaskService — list endpoints без проверки ownership

- `getAllByFolder` фильтрует по `folderId`, но не проверяет что folder принадлежит пользователю
- `getAllByGoal` аналогично
