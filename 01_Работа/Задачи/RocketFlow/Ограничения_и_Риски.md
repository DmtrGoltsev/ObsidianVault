# Ограничения и риски

Известные ограничения текущего состояния RocketFlow, требующие внимания.

## 1. Build-only CI (web и Android)

**Статус:** активно. **Влияние:** среднее.

Web и Android CI проверяют только сборку, но не запускают runtime-тесты. Регрессии в логике могут пройти CI незамеченными.

См. [[Задача_CI_Runtime_Lanes]].

## 2. Transactional save path

**Статус:** мониторинг. **Влияние:** низкое.

Операции сохранения (создание/обновление задач, целей, папок) выполняются в транзакциях. При высоких нагрузках возможны конфликты [[Optimistic_Locking|оптимистичной блокировки]].

## 3. Notification horizontal hardening

**Статус:** мониторинг. **Влияние:** среднее.

Нотификации проверены локально ([[Док_Нотификации_E2E]]), но:
- Не проверены под нагрузкой (множество одновременных напоминаний)
- Не проверены с реальным FCM на staging ([[Задача_Staging_Notification_Cert]])
- [[PostgreSQL_Advisory_Lock]] защищает от дублей, но однопоточная модель — узкое место

## 4. Android feature parity

**Статус:** мониторинг. **Влияние:** среднее.

Не все фичи веб-клиента реализованы в Android:
- Некоторые расширенные настройки планирования
- Возможно, отставание по UI-компонентам

Требуется сверка с [[MOC_Android]] и [[MOC_Веб]].

## 5. Backend prod deploy blocked

**Статус:** блокер. **Влияние:** высокое.

Prod deploy preflight 2026-06-08 остановлен без деплоя. Причины: canonical workflow deploy branches `MVP2`/`release_1`, текущая локальная ветка `MVP3`; readiness secrets/env не доказан из shell (`HEXCORE_PROD_SSH_*`, `ROCKETFLOW_PROD_BACKUP_*` отсутствуют); backup user mismatch — preferred `rocketbackup`, ignored local config points to `rocketdeploy`.

Нужны явные решения пользователя: target ref/method (`workflow_dispatch` vs merge/push to `MVP2`/`release_1`), backup expectation, secrets/env readiness. См. [[Док_Prod_Deploy_State]].
