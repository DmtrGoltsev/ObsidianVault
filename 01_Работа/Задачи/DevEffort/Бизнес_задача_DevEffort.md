# Бизнес-задача DevEffort

Создать portable JS-приложение для оценки трудозатрат IT-продуктов по релизам. Два варианта поставки: HTML в браузере (desktop) и Android PWA.

## Разложение доработки

Каждая доработка в релизе состоит из двух частей:

1. **ЕФС** — экранная/фронтовая часть: workflow, экраны, новая ФП, интеграции.
2. **Бэк** — backend-часть: проектирование, БД, интеграции/API, тестирование, развертывание, НФТ.

## Модель расчёта

### ЕФС — входы

| Поле | Тип | Описание |
|------|------|----------|
| newFeature | bool | Новая ФП (каркас) |
| screens | number | Количество добавленных/изменённых экранов |
| printedForms | number | Количество печатных форм |
| integrations | number | Количество интеграций (добавление/изменение) |
| fpConfirmation | bool | Добавляется ФП Подтверждение |
| fpFraudMonitoring | bool | Добавляется Фрод Мониторинг |
| useWorkflow | bool | Используется workflow |
| hasOtherModules | bool | Есть смежные доработки других модулей (отладка +20%) |

### ЕФС — формулы

```
workflowCount    = IF(useWorkflow, MAX(1, FLOOR(screens / 5)), 0)
restCount       = IF(useWorkflow, 0, screens)

totalDev        = screens * 0.5 + workflowCount * 1
                + IF(newFeature, 4, 0) + restCount * 1

debugging       = IF(hasOtherModules, totalDev * 0.2, 0)
```

**По этапам (чистые):**

| Этап | Формула |
|------|---------|
| Аналитика | totalDev / 3 |
| Разработка | totalDev |
| Отладка | debugging (= totalDev * 0.2, если hasOtherModules) |
| ИФТ | totalDev |
| Фронт раз | screens * 0.5 + restCount * 1 |
| ПСИ+НТ | hotfix ? 1 : 3 |
| Тираж (Внедрение) | hotfix ? 1 : 2 |

**С риском 35%:** все этапы `*= 1.35`. **Итого ч-д** = `SUM(все этапы)`. **пше/год** = `итого / 217`.

### Бэк — входы

**Общий параметр:** критичность системы = `MC+` | `MC` | `BC` | `ниже` (на весь релиз).

| Секция | Поля |
|--------|------|
| Проектирование | Проработка КАФО (bool), Работы в МЕТА (bool) |
| БД | Проектирование модели данных (bool + сложность), Создание таблиц (число + сложность), Миграция данных (bool + сложность), Скрипт (число + сложность) |
| Интеграции | Создание API/запись в очередь (число + сложность), Вызов API/чтение из очереди (число + сложность), Бизнес ошибки (bool + сложность) |
| Тестирование | Unit тесты (bool), Интеграционные тесты (bool), Нагрузочное тестирование (число), Функц. тестирование (bool + сложность) |
| Развертывание | Новая проектная область (bool), Труба (bool), ТУЗы (bool), Инфраструктура (bool) |
| НФТ | Безопасность (bool), ЖАМ (bool), Доступность (bool), Нагрузка (bool) |

**Сложность:** `просто` (x1) / `средне` (x1.5) / `сложно` (x2).

Для Нагрузочного тестирования и НФТ-пунктов сложность автовычисляется от критичности:
- `MC`/`MC+` → сложно
- `BC` → средне
- ниже → легко

### Бэк — ключевые формулы

```
БД_итого         = modelData + tables + migration + scripts
migration        = IF(enabled, scriptsResult * multiplier_by_complexity, 0)

Интеграции_итого = createAPI + readAPI + errors
errors           = IF(enabled, (createAPI + readAPI) * 0.2, 0)

Тестирование:
  unitTests      = IF(enabled, Интеграции_итого * 0.15, 0)
  intTests       = IF(enabled, Интеграции_взаимодействия * 0.2, 0)
  loadTests      = count * complexityMultiplier
  funcTests      = IF(enabled, (БД_итого + Интеграции_итого) * 0.4, 0)

Развертывание: каждый пункт bool → фиксированное ч/д (1, 2, 1, 2)

НФТ: каждый пункт bool → множитель (1.05–1.2) в зависимости от сложности

Бэк_итого_без_НФТ = Проектирование + БД + Интеграции + Тестирование + Развертывание
Бэк_итого          = Бэк_итого_без_НФТ * Безопасность * ЖАМ * Доступность * Нагрузка
```

**По этапам (чистые):**

| Этап | Формула |
|------|---------|
| Аналитика | Разработка / 3 + Проектирование |
| Разработка | (БД + Интеграции) * НФТ_коэффициенты |
| ИФТ | Тестирование + (БД + Интеграции) * 0.1 + Развертывание * 0.1 |
| ПСИ | 2 |
| Внедрение | = ПСИ |

**С риском 35%:** `*= 1.35`. **пше/год** = `/ 217`.

## Тип релиза

Тип определяется **автоматически** по объёму:

```
coreSum = SUM(Аналитика + Разработка + Отладка + ИФТ) по всем доработкам (без риска 35%)
isHotfix = (coreSum <= 5)
```

- **Hotfix** (coreSum ≤ 5): ПСИ+НТ = 1 ч-д, Внедрение = 1 ч-д.
- **Обычный релиз** (coreSum > 5): ПСИ+НТ = 3 ч-д, Внедрение = 2 ч-д.

ПСИ+НТ и Внедрение задаются **один раз на весь релиз** (не на доработку).

## Агрегация релиза

Все доработки суммируются:
- **Итого ч-д (ЕФС)** = SUM(все доработки.ЕФС.итого)
- **Итого ч-д (Бэк)** = SUM(все доработки.Бэк.итого)
- **Итого ч-д (релиз)** = ЕФС + Бэк + ПСИ+НТ + Внедрение

## Definition of Done

- Пользователь может создать релиз, указать тип (hotfix/обычный), критичность БЭК.
- Пользователь может добавить несколько доработок в релиз.
- Каждая доработка хранит разделение на ЕФС и Бэк.
- Расчёт воспроизводит логику Excel-калькулятора.
- Два варианта поставки: HTML в браузере + Android PWA.
- Риск 1.35 и делитель 217 — константы MVP.
