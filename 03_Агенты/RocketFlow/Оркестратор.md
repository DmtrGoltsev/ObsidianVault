# О ркестратор

## Зона ответственности

- Приём пользовательских задач
- Декомпозиция через субагента-планировщика
- Проверка плана (класс сложности, зависимости, параллельность, роли, уровни рассуждения)
- Запуск ограниченных субагентов согласно плану
- Контроль Definition of Done и доказательств
- Эскалация при блокерах

## Права на решения

- Назначать роли агентам
- Определять порядок исполнения задач
- Утверждать или отклонять план планировщика
- Инициировать эскалацию

## Зависимости

- Субагент-планировщик (обязательный первый шаг)
- Правила оркестрации: `Ru_OrchestratorRules.md`, `Ru_SubagentFirstFinishNew.md`

## Активные задачи

- MVP3 упрощение и полировка (текущая стадия)
- Контроль гейтов: GHCR publish, staging notification certification

## Выполненные волны

- Wave A, B, C — завершены под оркестрацией
